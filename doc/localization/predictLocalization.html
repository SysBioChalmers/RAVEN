predictLocalization

PURPOSE

predictLocalization Assign reactions to compartments using localization predictors.

SYNOPSIS

function [outModel, geneLocalization, transportStruct, scores,removedRxns] = predictLocalization(model, GSS,defaultCompartment, transportCost, maxTime, plotResults)

DESCRIPTION 

 predictLocalization  Assign reactions to compartments using localization predictors.

 Tries to assign reactions to compartments in a manner that is in
 agreement with localization predictors while at the same time maintaining
 connectivity.

 Parameters
 ----------
 model : struct
     a model structure. If the model contains several compartments they
     will be merged.
 GSS : struct
     a gene scoring structure as from parseScores.
 defaultCompartment : char
     transport reactions are expressed as diffusion between the
     defaultCompartment and the others. This is usually the cytosol. The
     default compartment must have a match in GSS.
 transportCost : double, optional
     the cost for including a transport reaction. If this is a scalar then
     the same cost is used for all metabolites. It can also be a vector of
     costs with the same dimension as model.mets. Note that negative costs
     will result in transport of the metabolite being encouraged (default
     0.5).
 maxTime : double, optional
     maximum optimization time in minutes (default 15).
 plotResults : logical, optional
     true if the results should be plotted during the optimization
     (default false).

 Returns
 -------
 outModel : struct
     the resulting model structure.
 geneLocalization : struct
     structure with the genes and their resulting localization.
 transportStruct : struct
     structure with the transport reactions that had to be inferred and
     between which compartments.
 scores : struct
     structure that contains the total score history together with the
     score based on gene localization and the score based on included
     transport reactions.
 removedRxns : cell
     cell array with the reaction ids that had to be removed in order to
     have a connected input model.

 Notes
 -----
 This function requires that the starting network is connected when it is
 in one compartment. Reactions that are unconnected are removed and saved
 in removedRxns. Try running fillGaps to have a more connected input model
 if there are many such reactions. The input model should also not include
 any exchange, demand or sink reactions, otherwise this function would not
 provide any results.

 In the final model all metabolites are produced in at least one reaction.
 This does not guarantee a fully functional model since there can be
 internal loops. Transport reactions are only included as passive diffusion
 (A <=> B).

 The score of a model is the sum of scores for all genes in their assigned
 compartment minus the cost of all transport reactions that had to be
 included. A gene can only be assigned to one compartment. This is a
 simplification to keep the problem size down. The problem is solved using
 simulated annealing.

 Examples
 --------
     [outModel, geneLocalization, transportStruct, scores, removedRxns] = ...
         predictLocalization(model, GSS, defaultCompartment, ...
             transportCost, maxTime, plotResults);

0795 function [S, g2c]=moveGene(S,model,g2c,geneToMove,toComp,nRxns,nMets)
0796 %Find the current compartment and update to the new one
0797 currentComp=find(g2c(geneToMove,:));
0798 g2c(geneToMove,:)=false;
0799 g2c(geneToMove,toComp)=true;
0800 
0801 %Find the reactions in the original model that the gene controls
0802 [I, ~]=find(model.rxnGeneMat(:,geneToMove));
0803 
0804 %Calculate their current positions in the S matrix
0805 oldRxns=I+(currentComp-1)*nRxns;
0806 
0807 %And their new positions
0808 newRxns=I+(toComp-1)*nRxns;
0809 
0810 %The metabolite ids also have to be changed in order to match the new
0811 %compartment
0812 metChange=nMets*(toComp-currentComp);
0813 
0814 %Update the reactions
0815 [I, J, K]=find(S(:,oldRxns));
0816 I=I+metChange;
0817 
0818 %Move the reactions
0819 S(:,oldRxns)=0;
0820 S(sub2ind(size(S),I,newRxns(J)))=K;

0823 %Finds which metabolites are unconnected, in the sense that they are never
0824 %a product or only a product in a reversible reaction where one reactant is
0825 %only a product in the opposite direction of that reaction. This function
0826 %ignores exchange metabolites. Returns a vector of metabolite indexes.
0827 %metsToCheck is an array of metabolite indexes to check for connectivity.
0828 %If not supplied then all metabolites are checked
0829 function unconnected=findUnconnected(S,nEM,metsToCheck)
0830 if nargin>2
0831     %Do this by deleting everything from the network that is not in
0832     %metsToCheck and that is not exchange metabolites
0833     I=false(size(S,1),1);
0834     I(1:nEM)=true;
0835     I(metsToCheck)=true;
0836     S=S(I,:);
0837 end
0838 
0839 em=false(size(S,1),1);
0840 em(1:nEM)=true;
0841 
0842 %Construct a matrix in which the reversible reactions are inverted
0843 I=sum(S>2,1) | sum(S>2,1);
0844 revS=S;
0845 revS(:,I)=revS(:,I)*-1;
0846 
0847 %First calculate the ones that are ok
0848 %Produced in 2 rxns, is exchange, is not used at all, is produced in
0849 %non-reversible, involved in more than 1 reversible reactions
0850 connected=sum(S>0,2)>1 | em | sum(S~=0,2)==0 | sum(S(:,~I)>0,2)>0 | sum(S(:,I)~=0,2)>1;
0851 
0852 %Then get the ones that are unconnected because they are never produced
0853 unconnected=sum(S>0 | revS>0,2)==0 & connected==false;
0854 
0855 %Then get the ones that are potentially unconnected
0856 maybeUnconnected=~connected & ~unconnected;
0857 %maybeUnconnected=find(maybeUnconnectedS);
0858 
0859 %The metabolites in maybeUnconnected are involved in one reversible
0860 %reaction and not produced in any other reaction. This means that the
0861 %reactions which have at least one met in maybeUnconnected as reactant and
0862 %one as product are unconnected. The metabolites in maybeUnconnected that
0863 %are present in those reactions are then dead ends
0864 deadRxns=any(S(maybeUnconnected,:)>0) & any(S(maybeUnconnected,:)<0);
0865 
0866 %Get the mets involved in any of those reactions
0867 problematic=any(S(:,deadRxns)~=0,2);
0868 
0869 %If any of these are in the maybeUnconnected list then the metabolite is
0870 %unconnected
0871 unconnected(problematic & maybeUnconnected)=true;
0872 
0873 %Map back to metsToCheck

0876 else

0878 end

0881 %Given a set of unconnected metabolites, this function tries to move each
0882 %gene that could connect any of them, calculates the number of newly
0883 %connected metabolites minus the number of newly disconnected metabolites.
0884 %As some metabolites are very connected, only 25 genes are checked. Genes
0885 %that have a low score in their current compartment are more likely to be
0886 %moved
0887 function [geneIndex, moveTo, deltaConnected, deltaScore]=selectGenes(S,nEM,nMets,nER,nRxns,model,unconnected,g2c,GSS)
0888 %If moveTo is 0 then the gene cannot connect any of the metabolites
0889 moveTo=zeros(numel(model.genes),1);
0890 deltaConnected=zeros(numel(model.genes),1);
0891 
0892 %First get where the metabolites are now
0893 nComps=size(g2c,2);
0894 comps=ceil((unconnected-nEM)/((size(S,1)-nEM)/nComps));
0895 
0896 %Find the corresponding metabolite indexes if they all were in the default
0897 %compartment
0898 dcIndexes=unique(unconnected-(comps-1)*nMets);
0899 
0900 %Then find them if they were in any other compartment
0901 allIndexes=dcIndexes;
0902 for i=1:nComps-1
0903     allIndexes=[allIndexes;dcIndexes+nMets*i];
0904 end
0905 
0906 %Also check which reversible reactions that could be used
0907 I=sum(S>2,1) | sum(S>2,1);
0908 revS=S;
0909 revS(:,I)=revS(:,I)*-1;
0910 
0911 %Find all reactions that could make any of the unconnected metabolites in
0912 %some other compartment
0913 newMets=setdiff(allIndexes,unconnected);
0914 [~, potential]=find(S(newMets,:)>0 | revS(newMets,:)>0);
0915 potential(potential<=nER | potential>nER+nRxns*nComps)=[]; %No exchange rxns or transport rxns
0916 
0917 %Map J to the real metabolic reactions in model
0918 rxnComps=ceil((potential-nER)/(nRxns));
0919 
0920 %Find the corresponding reaction indexes if they all were in the default
0921 %compartment
0922 dcRxnIndexes=potential-(rxnComps-1)*nRxns;
0923 
0924 %Get the genes for those reactions
0925 genes=find(sum(model.rxnGeneMat(dcRxnIndexes,:)>0,1));
0926 
0927 %For some cases there can be very many reactions to connect something. This
0928 %is in particular true in the beginning of the optimization if, say, ATP is
0929 %unconnected. Therefore limit the number of genes to be checked to 25.
0930 %Weigh so that genes with bad scores in their current compartment are more
0931 %likely to be moved.
0932 
0933 %Get scores for these genes
0934 [~, J]=find(g2c(genes,:));
0935 
0936 %Add a small weight so that genes in their best compartment could be moved
0937 %as well
0938 geneScores=GSS.scores(sub2ind(size(g2c),genes(:),J));
0939 modGeneScores=1.1-geneScores;
0940 if numel(genes)>25
0941     rGenes=genes(randsample(numel(genes),min(numel(genes),25),true,modGeneScores));
0942     
0943     %The sampling with weights could give duplicates
0944     rGenes=unique(rGenes);
0945     
0946     %Reorder the geneScores to match
0947     [~, I]=ismember(rGenes,genes);
0948     geneScores=geneScores(I);
0949     genes=rGenes;
0950 end
0951 for i=1:numel(genes)
0952     %Since one gene is moved at a time, only metabolites involved in any of
0953     %the reactions for that gene can become unconnected. This helps to
0954     %speed up the algorithm. First get all involved reactions in the
0955     %default compartment
0956     rxns=find(model.rxnGeneMat(:,genes(i)));
0957     
0958     %Then get their mets
0959     mets=find(sum(model.S(:,rxns)~=0,2)>0);
0960     
0961     %Then get their indexes in all compartments
0962     allIndexes=mets;
0963     for j=1:nComps-1
0964         allIndexes=[allIndexes;mets+nMets*j];
0965     end
0966     
0967     %Check which of the unconnected metabolites that these reactions
0968     %correspond to. This could have been done earlier, but it is fast. The
0969     %reversibility check is skipped because it is unlikely to be an issue
0970     %here. Worst case is that the gene is tested once to much
0971     [I, ~]=find(model.S(:,rxns));
0972     moveToComps=unique(comps(ismember(dcIndexes,I)));
0973     
0974     %Try to move the gene to each of the compartments
0975     bestMove=-inf;
0976     bestComp=[];
0977     for j=1:numel(moveToComps)
0978         newS=moveGene(S,model,g2c,genes(i),moveToComps(j),nRxns,nMets);
0979         
0980         %Check how many metabolites that are unconnected after moving the
0981         %gene
0982         dConnected=numel(unconnected)-numel(findUnconnected(newS,nEM,[allIndexes;unconnected]));
0983         if dConnected>bestMove
0984             bestMove=dConnected;
0985             bestComp=moveToComps(j);
0986         end
0987     end
0988     
0989     %Add the difference in connectivity and where the genes should be moved
0990     moveTo(genes(i))=bestComp;

0992 end
0993 
0994 %Finish up
0995 geneIndex=genes(:);
0996 moveTo=moveTo(geneIndex);
0997 deltaConnected=deltaConnected(geneIndex);

0999 end
